Setup Guide

環境セットアップガイド

Google Cloud が初めての方でも、手順通りに進めれば 15 分程度で完了します。

1 Google アカウントの準備

Google Cloud を利用するには Google アカウントが必要です。

  • Gmail アカウントをお持ちの方 → そのまま使えます
  • 大学のメールアドレスが Google Workspace の場合 → そのまま使えます
  • Google アカウントをお持ちでない方 → accounts.google.com で無料作成

2 Google Cloud プロジェクトの作成

  1. console.cloud.google.com にアクセスしてログイン
  2. 画面上部の「プロジェクトの選択」をクリック → 「新しいプロジェクト」
  3. プロジェクト名: handson-XXXX（XXXX は好きな名前）を入力して「作成」
  4. 通知から「プロジェクトを選択」で切り替え

課金（Billing）の設定

バウチャーをお持ちの方（当日配布）:

  1. バウチャーコードを手元に用意
  2. cloud.google.com/billing/coupon を開く
  3. コードを入力して「同意して続行」

無料トライアルの方: コンソール上部の「無料で利用開始」から登録（$300分の無料クレジット）

3 BigQuery API の有効化

コンソール上部の検索バーに BigQuery API と入力 → 選択 → 「有効にする」をクリック

4 Analytics Hub でデータをサブスクライブ

ここが最も重要なステップです。運営側が提供するデータを、自分のプロジェクトから参照できるようにします。

  1. 左メニュー →「Analytics Hub」をクリック
  2. 「検索」タブで サンエー または sana と入力して検索
  3. 表示されるリスティングをクリック →「サブスクライブ」
  4. プロジェクト: 自分のプロジェクトを選択、データセット名: meetup8、ロケーション: asia-northeast1
  5. 2つ目以降は「既存のデータセットを使用」→ meetup8 を選択

以下の 全5つ のリスティングをサブスクライブしてください:

リスティング名内容
card_master - カードマスターポイントカード情報（匿名化済み）
customer_master - 顧客マスター顧客属性情報（個人情報除外済み）
customer_sales_details - 売上明細購買トランザクション（原価除外済み）
shop_item_daily_performance - 店別日別実績店舗 × 商品 × 日の実績データ
food_item_master - 商品マスター食品商品情報（原価除外済み）

ハンズオンコンテンツ

「沖縄のスーパーのデータを AI × Google Cloud で分析してみよう」

1 環境セットアップ & まずデータを見てみよう（15分）

分析環境に慣れ、データの全体感をつかみます。

-- まずは売上明細を見てみよう
SELECT * FROM `meetup8.customer_sales_details` LIMIT 10;

-- このデータは全部で何件？
SELECT COUNT(*) AS total_records FROM `meetup8.customer_sales_details`;
-- → 3,059件。「1つ1つが、誰かがサンエーでレジを通した記録です」

2 AI に聞いてみよう — Gemini で自然言語分析（30分）

SQLを知らない初学者でも「日本語で聞けば答えが返ってくる」体験をします。

Phase 1: まず AI に聞いてみる

Gemini in BigQuery で以下のような質問を試してみましょう:

  • 「このデータの中で、一番高い買い物はいくらでしたか？」
  • 「店舗ごとの売上合計を、多い順に並べてください」
  • 「1月で一番売上が高かった日はいつですか？その理由として考えられることは？」

Phase 2: AI が生成した SQL を読み解く

-- 例: Gemini が生成した「店舗別売上合計」のSQL
SELECT
  store_code,
  SUM(tax_included_amount) AS total_sales
FROM `meetup8.customer_sales_details`
GROUP BY store_code
ORDER BY total_sales DESC;

SQL の 4 つの基本要素: SELECT（何を見たいか）、FROM（どのテーブルから）、GROUP BY（何でまとめるか）、ORDER BY（どう並べるか）

Phase 3: 自分の問いを AI にぶつける

  • 「一番たくさんの商品を一度に買った人は、何品買った？」
  • 「特定の商品コードを買った人は、他にどんな商品を買っている？」
  • 「金曜日と日曜日で、買い物の仕方に違いはある？」
  • 自由に好きな質問を考えてOK！

3 謎解き分析 —「答えのある問い」に挑戦（30分）

データ分析の本質は「なぜ？」を考えること。AI に聞いても良いし、SQL を自分で書いても良い。手段は問いません。

🔎 Q1: 元旦の売上が突出して高いのはなぜ？

SELECT
  usage_date,
  COUNT(DISTINCT CONCAT(card_no, store_code, trans_no)) AS txn_count,
  SUM(tax_included_amount) AS total_sales,
  SAFE_DIVIDE(SUM(tax_included_amount),
    COUNT(DISTINCT CONCAT(card_no, store_code, trans_no))) AS avg_per_txn
FROM `meetup8.customer_sales_details`
GROUP BY usage_date
ORDER BY usage_date;

答え: 1月1日は売上64,037円で月間最高。客単価3,370円も最高。お正月の買い出し（おせち材料・お年始用品）で1回あたりの購入額が跳ね上がる。

🔎 Q2: 1月28日（土曜日）の売上が極端に低いのはなぜ？

SELECT
  FORMAT_DATE('%A', PARSE_DATE('%Y%m%d', usage_date)) AS day_of_week,
  SUM(tax_included_amount) AS total_sales,
  COUNT(*) AS item_count
FROM `meetup8.customer_sales_details`
GROUP BY day_of_week
ORDER BY total_sales DESC;

答え: 日曜日が189,006円で最も高く、土曜日は111,404円と意外と低い。特に1/28(土)は月間最低の12,147円。前日に月間最多の23取引があり、金曜に買い溜めした翌土曜は来店が減るパターン。
「土曜が一番売れるはず」という思い込みが、データで覆される瞬間。これがデータ分析の醍醐味。

🔎 Q3: 一番大量に買い物をしたお客さんはどんな買い方？

SELECT
  card_no, store_code, usage_date,
  COUNT(*) AS item_count,
  SUM(tax_included_amount) AS basket_total
FROM `meetup8.customer_sales_details`
GROUP BY card_no, store_code, trans_no, usage_date
ORDER BY item_count DESC
LIMIT 5;

答え: 最大バスケットは1/19（木）38品・9,756円（1品平均256円）。最高額バスケットは1/1（元旦）37品・17,349円。品数は近いのに金額は1.8倍 — 正月は単価の高い商品を買っている。

🔎 Q4（発展）: ポイントを多く持つ人は本当にたくさん買い物をしている？

SELECT
  c.card_no,
  c.total_point AS current_points,
  c.cumulative_issued_point AS lifetime_points,
  COUNT(*) AS purchase_count,
  SUM(s.tax_included_amount) AS total_spent
FROM `meetup8.card_master` c
JOIN `meetup8.customer_sales_details` s ON c.card_no = s.card_no
GROUP BY c.card_no, c.total_point, c.cumulative_issued_point
ORDER BY total_spent DESC
LIMIT 10;

答え: 必ずしも一致しない。ポイント残高が多い人は「貯める派」、少ない人は「使う派」。顧客行動の多様性がデータから見える。

セキュリティ多層防御

1

ソースデータ匿名化

個人情報 → HEX エンコード / 原価 → マスク（*****）/ カード番号 → SHA-256 ハッシュ

2

ビューによるカラム制御

HEX カラム・原価カラムを除外 / 店舗コードを店舗名に変換

3

エグレス制御（DCR 機能）

EXPORT / COPY / CREATE TABLE AS SELECT を全てブロック

4

IAM アクセス制御

Google Group メンバーのみサブスクライブ可能

5

プロジェクト分離

プロバイダーとサブスクライバーを分離 / クエリコストは参加者負担

操作許可/ブロック

Gemini in BigQuery（自然言語 → SQL）許可

FAQ

よくあるトラブルと対処法

「課金アカウントが設定されていません」と表示される
課金設定が完了していません。バウチャーコードの入力、または無料トライアルの登録を行ってください。
Analytics Hub でリスティングが見つからない
検索キーワード（サンエー または sana）が正しいか確認。ハンズオン用 Google アカウントでログインしているか確認。それでも見つからない場合は運営スタッフにお声がけください。
サブスクライブ時に「権限がありません」と表示される
自分のプロジェクトが選択されているか、課金アカウントがリンクされているか、BigQuery API が有効化されているか確認してください。
クエリ実行時に「アクセスが拒否されました」と表示される
サブスクライブが正しく完了していない可能性があります。セットアップの Step 4 を再度確認してください。
データをダウンロードしようとしたらブロックされた
正常な動作です。データクリーンルームの Egress 制御により、データのダウンロード・エクスポートは意図的に防止されています。分析結果は画面上で確認してください。
プロジェクト ID が「既に使用されています」と表示される
プロジェクト ID は世界で一意です。handson-tanaka-0315、handson-ryudai-2026 のようにユニークな名前にしてください。
Gemini がデータセットを認識しない
左パネルで meetup8 データセットをクリックして選択した状態で再度質問してみてください。
